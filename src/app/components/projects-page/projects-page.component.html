This was a website I made as part of a college assignment. It served as a portfolio to showcase my coding skills at the time as well as a place for me to store business contacts. This was the very first website I created which used REST and had a proper backend for storing data. A very standard approach to create web pages was used in this project, in the sense that no single page application framework (like Angular or React) was used, but instead routing was done through GET and POST requests. Even though this was a very tradicional (and somewhat outdated) way of creating websites, it served as a good exercise for me to understand and practice the basis of full-stack web development.

Technologies used: REST, Node, ExpressJS, PassportJS, EJS, MongoDB

The Bracketeams tournament website is a brackets website (similar to challonge.com) that I've done in conjunction with 5 other individuals as a lead software developer. This project was my first group experience in a software development environment, and it was quite challenging. We used the agile methodology, as well as Jira in oder to coordinate the workflow. For the front-end we used Angular, and for the back-end we used ExpressJS and MongoDB. Authentication and authorization was done using JWT.

Technologies used: REST, ExpressJS, MongoDB, JWT, Angular
